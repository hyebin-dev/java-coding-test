# 제출 전 체크리스트

제출 직전에 10~20초만 보고 실수로 틀리는 것을 막는다.

## 입력/출력
- 입력이 많으면 BufferedReader + StringTokenizer(또는 커스텀 FastScanner)로 바꿨나?
- 출력이 많으면 StringBuilder로 모아서 한 번에 출력하나?
- 공백/줄바꿈 포맷 요구사항(마지막 공백, 마지막 줄바꿈) 정확한가?

## 자료형/오버플로우
- 합/곱/거리/누적값에 int 오버플로우 가능성 없나? (애매하면 long)
- (a*b) 같은 곱셈은 long 캐스팅을 먼저 했나? ( (long)a * b )

## 인덱스/경계
- 0-index / 1-index 섞이지 않았나?
- for 범위(<= / <) 경계값 정확한가?
- 빈 배열/원소 1개/최솟값/최댓값 케이스 고려했나?

## 시간복잡도/자료구조 선택
- N, M 크기 보고 O(N^2) / O(N^3) 터지지 않나?
- HashMap/HashSet 필요할 때 List로 선형 탐색하고 있지 않나?
- 정렬이 필요한데 정렬을 안 했거나, 불필요한 정렬을 반복하고 있지 않나?

## BFS/DFS
- visited 처리 시점(큐에 넣을 때 vs 꺼낼 때) 일관되게 했나?
- DFS 재귀면 스택오버플로우 가능성 없나? (필요하면 iterative)
- 그래프가 방향/무방향인지 간선 추가를 양쪽으로 했나?

## DP
- dp 초기값/불가능 상태(INF) 설정이 맞나?
- 점화식에서 이전 상태 인덱스가 음수가 되지 않나?
- 모듈러 문제면 매 연산마다 mod 적용했나?

## 우선순위큐/다익스트라
- 거리 배열 초기값을 충분히 큰 값으로 뒀나?
- (dist > curDist) 스킵 조건 넣었나?
- 간선 가중치 음수면 다익스트라 쓰면 안 됨(벨만포드/다른 접근)

## 이분탐색
- low/high 초기값이 문제 정의와 맞나?
- mid 계산에서 무한루프 방지(while (lo <= hi), 업데이트 방향) 맞나?
- "가능/불가능" 판정 함수(check)가 단조성(모노토닉) 만족하나?

## 구현/시뮬레이션
- 상태 업데이트 순서 때문에 이전 상태가 덮어써지지 않나?
- 동시에 바뀌어야 하는 건 임시 배열/임시 리스트를 썼나?

## 최종 확인(가장 중요)
- 예제 입력 직접 돌려서 예제 출력과 완전히 일치하나?
- 내가 만든 반례 1~2개로 다시 확인했나?
